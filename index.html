Потоки

Так как про определение потока как последовательности событий во времени можно и так прочитать в любой статье, посвященной RxJS, данная презентация расскажет о потоках на примере:

Красная Шапочка и служба промисов

Красная шапочка решает использовать промисы, чтобы отправить бабушке пирожок. 

          sendPie()
            .then(grandmaEatsPie)
            .catch(pieIsEatenByWolf);

Красная Шапочка и суровая реальность

На самом деле задача отправки пирожков немного сложнее:

  • Отправлять пирожки бабуле нужно по её звонку;
  • Выпекание пирожка требует времени (разного, в зависимости от настроения и навыков Красной Шапочки);
  • Выпекать надо, когда есть запрос (все хотят свеженький пирожочек!);
  • Отправленный пирожок может добраться до бабули, а может быть нагло съеден волком.
          phone.addEventListener(
            'callFromGrandma',
            ( ) =>
              bakePie()
                .then(sendPie)
                .then(grandmaGetsPie)
                .catch(eatenByWolf),
          );

Красная шапочка и еще более суровая реальность

Проблемы данного кода:

  • Бабушка может звонить несколько раз подряд;
  • Пирожок нельзя переслать, если он попал к волку;
  • Пирожки выпекаются и отправляются по отдельности;
  • Порядок отправки и выпекания пирожков случаен;
  • Если пирожки не случились - надо заказать суши; 
            phone.addEventListener(
             'callFromGrandma',
             () =>
               bakePie()
                 .then(sendPie)
                 .then(grandmaGetsPie)
                 .catch(eatenByWolf),
            );

Красная Шапочка задумывается об отправке пирожков как о потоке.

Красная шапочка классификацирует Observable

Холодные (Cold)

Не отдают значений, пока на них нет подписки, т.к. по-настоящему ленивы.

Каждая подписка создает незавимый поток, который отдает все события с начала (у всех подписок одинаковый набор событий).

              of(...)
            
              from(...)
            
              timer(...)
            
              inverval(...)

Горячие (Hot)

Полученные подписчиками события зависят от времени подписки.

События, происходящие, когда подписки нет, игнорируются. Всем подписчикам отдается одно и то же событие.

              fromPromise(...)
            
              fromEvent(...)

Hot vs Cold

Cold.

          const A = interval(5);
          
          A.pipe(take(2)).subscribe(console.log);
          
          setTimeout(() => {
            console.log('timeout');
            A.pipe(take(2)).subscribe(console.log);
          }, 1500);

Консоль.

0 1
timeout 0 1

Hot vs Cold

Hot.

          const A = fromEvent(document, 'mousemove');
          
          A.pipe(take(2))
            .subscribe(e => console.log(e.clientX));
          
           setTimeout(() => {
            console.log('timeout');
            A.pipe(take(2))
              .subscribe(e => console.log(e.clientX));
           }, 1500);

Консоль.

199 236
timeout 253 267

Красная шапочка отправляет пирожки с помощью RxJS

Работа с событиями напоминает работу с массивами. 

          fromEvent(phone, 'callFromGrandma').pipe(
            bufferTime(500),
            filter(hasCallsFromGrandma),
            concatMap(calls => of(calls).pipe(
              concatMap(bakePies),
              concatMap(sendPies),
              retry(3),
            )),
            catchError(orderSushi),
          ).subscribe(grandmaEats);
Создаем Observable Объединяем в "партии" Проверяем наличие запросов   Печем (асинхронно) Отправляем (асинхронно) Пересылаем при ошибке   Ошибка - заказ суши Успех - еда у бабушки

Работаем с массивами как с событиями

Поток. 

            from([1, 2, 3, 4])
              .pipe(
                map(x => x*3),
                filter(x => x % 2 === 0),
                toArray(),
              )
             .subscribe(console.log);

Цепочка операций над масивом. 

            const data = [1, 2, 3, 4]
              .map(x => x*3)
              .filter(x => x % 2 === 0);
            console.log(data);

Консоль.

[6, 12]

Особенности цепочки операций над массивом

Каждая операция создает копию.

          const data = [1, 2, 3, 4]
            // map creates a copy of initial array
            .map(x => x*3)
            // filter creates a copy of initial array
            .filter(x => x % 2 === 0);
          console.log(data);

Особенности работы с потоком

Элемент сразу проходит через все "операции". 

          from([1, 2, 3, 4])
            .pipe(
              tap(console.log),
              map(x => x*3),
              tap(console.log),
              filter(x => x % 2 === 0),
              tap(console.log),
              toArray(),
            )
           .subscribe(console.log);

Консоль.

1 3
2 6 6
3 9
4 12 12
[6, 12]

Отличие синхронных и асинхронных операций

Код, состоящий из сихронных операторов, всегда выполняется последовательно.

Синхронный Observable. 

          console.log('start');
          of(1)
            .pipe(switchMap(x => of(x*2)))
            .subscribe(x => console.log('result', x));
          console.log('end');

Консоль.

start result 2 end

Отличие синхронных и асинхронных операций

Наличие асинхронных операторов всегда приводит к выполнению кода (даже не содержащего реальных асинхронных действий) асинхронно.

Асинхронный Observable.

          console.log('start');
          of(1)
            .pipe(switchMap(
              x => fromPromise(Promise.resolve(x*2))
            ))
            .subscribe(x => console.log('result', x));
          console.log('end');

Консоль.

start end result 2

Сравнение синхронного и асинхронного кода

Синхронный Observable.

          of(1)
            .pipe(switchMap(
              x => of(x*2)
            ))
            .subscribe(x =>
              console.log('result', x));

Асинхронный Observable.

          of(1)
            .pipe(switchMap(
              x => fromPromise(
                Promise.resolve(x*2)
            )))
            .subscribe(x =>
              console.log('result', x));

Польза лени

RxJS использует ленивые вычисления: операторы не выполняются, когда у потока нет подписчиков.

Следующий код не вызовет переполнение памяти, т.к. у него нет подписчиков: 

          range(1, Number.POSITIVE_INFINITY)
           .pipe(tap(console.log));

Преимущество ленивых вычислений

Ленивые вычисления дают возможность работать с бесконечными потоками.

Подсчет факториала. 

          range(1, Number.POSITIVE_INFINITY)
           .pipe(
              take(5),
              reduce((acc, x) => acc * x, 1),
            )
            .subscribe(console.log);

Консоль.

120

Observable-ы и тленность бытия время жизни

Могут быть конечные и бесконечные.

Сразу завершится. 

          of(1)
            .subscribe(
              () => console.log('success'),
              () => console.log('error'),
              () => console.log('completed')
            );

Будет жить "вечно". 

          fromEvent(document, 'click')
            .subscribe(
              () => console.log('success'),
              () => console.log('error'),
              () => console.log('completed')
            );

Ошибка сразу завершит Observable, но не вызовет "onCompleted" (onCompleted" !== "Finally").

Завершенность Observable - возможность освободить ресурсы

Каким образом может завершиться?

Конечный набор событий.

  • of('something')
  • from([1, 2, 3])
  •                 fromPromise(promise)

Ошибка в процессе выполнения.

.catchError() не спасет от завершения.

            fromEvent(document,
              'click')
              .pipe(map(() => {
                throw new Error('err');
              }));

Операторы "принудильного завершения".

  • first
  • take
  • takeUntil
  • takeWhile
  • unsubscribe

Особенности принудительного завершения. first() !== take(1).

Оператор take не вызовет ошибку, если к моменту завершения не найдено нужного значения.

take(1) и пустой поток.

            empty()
              .pipe(take(1))
              .subscribe(
                () => console.log('ok'),
                () => console.log('fail'),
                () => console.log('done')
              );

Консоль.

done

Особенности принудительного завершения. first() !== take(1).

Оператор first вызовет ошибку, если значение не найдено.

first() и пустой поток.

            empty()
              .pipe(first())
              .subscribe(
                () => console.log('ok'),
                () => console.log('fail'),
                () => console.log('done')
              );

Консоль.

fail

Особенности принудительного завершения. first() !== take(1).

Оператор first дополнительно может принимать параметр-предикат.

first() с предикатом.

            from([1, 2, 3])
              .pipe(first(x => x > 5))
              .subscribe(
                () => console.log('ok'),
                () => console.log('fail'),
                () => console.log('done')
              );

Консоль.

fail

Unsubscribing

Не используйте unsubscribe сразу после subscribe.

Подписка будет отменена прежде, чем возникнут события. 

          fromEvent(document, 'click')
           .subscribe(console.log)
           .unsubscribe();

Бессмысленно, так как observable конечный и синхронный. 

          of(100500)
           .subscribe(console.log)
           .unsubscribe();

Unsubscribing

Правильный вариант с .unsubscribe(). 

          fromEvent(document, 'click')
           .subscribe(console.log)
           .unsubscribe();
        
          const customSubscription = 
            fromEvent(document, 'click')
            .subscribe(console.log);
          ...
          customSubscription.unsubscribe();

Комбинирование. withLatestFrom, combineLatest, forkJoin.

withLatestFrom применяется, когда при получении события надо взять "свежее" значение из другого потока.

Пример работы withLatestFrom. 

          const A = new BehaviorSubject(1);
          const B = new BehaviorSubject('a');
          
          A.pipe(withLatestFrom(B))
           .subscribe(console.log);
          
          A.next(2); B.next('b');
          A.complete(); B.complete();

Консоль.

[1, 'a'] [2, 'a']

Комбинирование. withLatestFrom, combineLatest, forkJoin.

combineLatest применяется, когда необходимо слушать каждое изменение в любом из потоков.

Пример работы combineLatest. 

          const A = new BehaviorSubject(1);
          const B = new BehaviorSubject('a');
          
          combineLatest(A, B)
           .subscribe(console.log);
          // A.pipe(combineLatest(B))
          
          A.next(2); B.next('b');
          A.complete(); B.complete();

Консоль.

[1, 'a'] [2, 'a'] [2, 'b']

Комбинирование. withLatestFrom, combineLatest, forkJoin.

forkJoin применяется, когда необходимо взять последнее значение из каждого потока по их завершении.

Пример работы forkJoin. 

          const A = new BehaviorSubject(1);
          const B = new BehaviorSubject('a');
          
          forkJoin(A, B)
           .subscribe(console.log);
          
          A.next(2); B.next('b');
          A.complete(); B.complete();

Консоль.

[2, 'b']

Неочевидные зависимости

Порядок может иметь значение для алгоритмов применения изменений. 

          const A = new BehaviorSubject(1);
          const B = new Subject();
           
          A.asObservable()
           .subscribe((val) => {
             console.log('A value is ', val);
             B.next(val * 10);
          });
          
          B.asObservable().pipe(
            withLatestFrom(A.asObservable())
          ).subscribe(([bVal, aVal]) => {
           console.log(`A is ${aVal}; B is ${bVal}`);
          });
          
          setTimeout(() => A.next(2));
A value is 1 A value is 2 A is 1; B is 20  

Асинхронность

Внесение асинхронности изменяет значения в потоках. 

          const A = new BehaviorSubject(1);
          const B = new Subject();
           
          A.asObservable()
           .subscribe((val) => {
             console.log('A value is ', val);
             B.next(val * 10);
          });
          
          B.asObservable().pipe(
           debounceTime(0),
            withLatestFrom(A.asObservable()),
          ).subscribe(([bVal, aVal]) => {
           console.log(`A is ${aVal}; B is ${bVal}`);
          });
          
          setTimeout(() => A.next(2));
A value is 1 A value is 2 A is 2; B is 20  

Изменение порядка

Меняя местами подписки, мы также меняем значение в потоках. 

          const A = new BehaviorSubject(1);
          const B = new Subject();
           
          B.asObservable().pipe(
              withLatestFrom(A.asObservable())
          ).subscribe(([bVal, aVal]) => {
           console.log(`A is ${aVal}; B is ${bVal}`);
          });
          
          A.asObservable()
           .subscribe((val) => {
             console.log('A value is ', val);
             B.next(val * 10);
          });
          
          setTimeout(() => A.next(2));
A value is 1 A is 1; B is 10 A value is 2 A is 2; B is 20
